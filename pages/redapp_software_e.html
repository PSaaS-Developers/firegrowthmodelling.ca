REDapp V6.2.4 (2019-01-06) - New release coming soon!

System Requirements for REDapp 6.2.4

 
            Minimum Java Version: Java SE 8 (Oracle preferred). Update 151 or better.

            Minimum OS Required:

            Windows 7 or later.

            Intel-based Mac OS X 10.8.3 (Mountain Lion) or later.

            Oracle Linux 5.5+, 6.x (32-bit), 6.x (64-bit).

            Red Hat Enterprise Linux 5.5+, 6.x (32-bit), 6.x (64-bit).

            Ubuntu Linux 12.04 LTS or later.

            Suse Linux Enterprise Server 10 SP2, 11.x.
 

Changes since the last release:

#367 The @ Hour for the Hourly (Van Wagner) HFFMC calculation method can now be specified when importing hourly weather in the Statistics tab. An option to use day 1 FFMC at hour 17 has also been provided.

#547 Typo fixed in the Assumptions dialog under General FWI System Assumptions.

#712 The FBP Calculator now uses accelerating rate of spread to calculate fire intensity, spread distances, elliptical fire area and perimeter. Previous versions of REDapp and other interpretations of the Canadian Fire Behavior Prediction System use equilibrium rate of spread for subsequent fire behavior calculations.

#785 The Map tab no longer uses Google map services. Background imagery is now sourced from Open Street Map. An option to source background imagery from a custom web map service has also been added.

#807 Terrain characteristics have been removed from the Settings dialog.

#809 A time zone override was removed from the Statistics tab.

#810 The percentile option for displaying ensemble forecasts in the Weather tab has been removed. The custom option has been integrated into the GEM Deterministic, GEM Ensemble, and NCEP Ensemble options.

#817 Elliptical fire perimeters displayed on the Map tab can now be exported as Shapefiles.

#824 Naming conventions for Weather tab exports now include the model (GEM or NCEP) and differentiate deterministic and control members from other ensemble members.

#831 A scale bar that responds the Unit setting (Metric or Imperial) has been added to the Map tab.

 

Bug fixes since the last release:

#712 Rounded values where being passed from the FWI Calculator to the FBP Calculator. Rounding is no longer applied to calculated values transferred to other tabs.

#714 Changing the coordinate unit setting did not change the format in the Latitude and Longitude fields. This has been fixed and the user interace better reflects the selected coordinate unit format.

#804 Ignition location coordinates are now saved on exit when the “Retain input values when exiting” setting is selected.

 

Known issues in this release:

#365 When importing an hourly weather file, the Daily display option in the Statistics tab returns 1 row of noon standard weather labeled 1300 hours when using Daylight Savings Time. However, the data associated with this 1300 hour record are actually the data from hour 0:00 in the import file.

#367 The new option to use day 1 FFMC at hour 17 when importing hourly weather and applying the Hourly (Van Wagner) HFFMC method does not adjust to standard time. This setting should apply day 1 FFMC to hour 16 when standard time is in effect.

#621 Default weather data is not labeled correctly when manually adding new data to an existing weather table. The precipitation column contains wind direction values and wind speed column appears to contain precipitation values.

#661 The Spotting Calculator label for Number of Trees Torching says: "Number Tree Torching".

#805 The default setting for “Retain input values when exiting” should be ON. Clicking the Restore Defaults button turns this setting off.
